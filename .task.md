## Approach
1. Download the latest sherpa-onnx AAR (`sherpa-onnx-1.12.29.aar`) from https://github.com/k2-fsa/sherpa-onnx/releases/tag/v1.12.29
2. Extract `jni/{abi}/libonnxruntime.so` and `jni/{abi}/libsherpa-onnx-jni.so` from the AAR
3. Replace prebuilts in `packages/sherpa-onnx.rn/prebuilt/android/{arm64-v8a,armeabi-v7a,x86_64}/`
4. Verify 16KB alignment on the new `.so` files using the ELF p_align check
5. Build sherpa-voice release AAB and confirm `libonnxruntime.so align=16384`

## Background
- Current prebuilts bundle onnxruntime v1.17.1 (4KB aligned — fails Play Store 16KB check)
- sherpa-onnx v1.12.29 ships with onnxruntime v1.23.2 which includes the upstream 16KB fix (PR #24947)
- Affected files: `packages/sherpa-onnx.rn/prebuilt/android/` and `packages/sherpa-onnx.rn/android/src/main/jniLibs/`
- Issue tracked: https://github.com/k2-fsa/sherpa-onnx/issues/2641
